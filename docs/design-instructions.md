# Sunoパ！作業開始パックツール 設計指示書

## 目的

Sunoパ！の記事作成、音源整理、SE_Pon登録、サムネ制作、SNS導線をCodexに依頼するとき、毎回必要な情報をひとまとめにした「作業開始パック」を作る。

このツールは、Radio-Article-Studioの補助機能として作っても、別ツールとして作ってもよい。目的は、Codexが迷わず同じ順番で作業できる状態を作ること。

## 作りたいもの

```text
Sunoパ！作業開始パックツール
├─ 放送回を選ぶ
├─ 記事タイプを選ぶ
├─ Codexへ渡す作業開始パックを生成
├─ 音源保存先を生成
├─ SE_Pon登録順を生成
├─ 楽曲紹介サムネの制作指示を生成
├─ WordPress/SNS/漫画の作業範囲を指定
└─ 作業後チェックリストを生成
```

## Radio-Article-Studioとの関係

Radio-Article-Studioは、ゲスト名、応募者名、楽曲タイトル、アーティスト名、楽曲URL、アイコン、サムネ素材などの正確情報を持つ正本として使う。

この作業開始パックツールは、Radio-Article-Studioから得た情報をCodex向けに並べ替えて、実作業に必要な形へ整える。

## 入力項目

### 基本情報

```text
放送回名:
放送日:
記事タイプ: ゲスト回 / リスナー応募楽曲オンエアー回
stand.fm URL:
Radio-Article-StudioのCodexパック:
スプレッドシートURLまたはCSVパス:
投稿先WordPressサイト:
WordPress方針: 下書きまで / 確認後公開 / すぐ公開
SNS・漫画: 作る / 作らない
```

### 音源整理

```text
音源整理: する / しない
音源ファイルの場所:
音源種別: 応募楽曲 / ゲスト楽曲 / 両方
保存先:
  応募楽曲: C:\Users\myabe\OneDrive\Desktop\ポン出し音源一覧\応募楽曲\{{放送回名}}\
  ゲスト楽曲: C:\Users\myabe\OneDrive\Desktop\ポン出し音源一覧\ゲスト楽曲\{{放送回名}}\
```

### SE_Pon

```text
SE_Pon登録: する / しない
登録先: 🎵 放送「曲・BGM」
全曲リピートON: はい
```

ゲスト回の基本順:

```text
1. ゲスト紹介曲1曲目
2. かなめ🦐の曲
3. べるぼ☂の曲
4. ゲスト紹介曲2曲目
5. 応募者楽曲（申し込み順）
```

応募者楽曲が多い場合は、ユーザーが「ここまで」と指定するか、Codexが登録前に確認する。

### 楽曲データ

各曲に次を持たせる。

```text
slot_no:
曲の種別: ゲスト曲 / かなめ🦐曲 / べるぼ☂曲 / 応募者曲
名前:
曲名:
AIアーティスト名:
楽曲URL:
URL種別: YouTube / Suno
音源ファイルパス:
アイコン画像パス:
楽曲サムネ取得元:
記事で触れるポイント:
X URL:
```

## 作業後チェックリスト

ツールは最後に、Codex用チェックリストも生成する。

```text
[ ] Radio-Article-Studioの情報を正本として確認
[ ] YouTube/Suno URLを埋め込み形式に変換
[ ] 音源を所定フォルダーに保存
[ ] SE_Ponの 🎵 放送「曲・BGM」 に順番通り登録
[ ] SE_Pon登録曲をすべてリピートON
[ ] 記事に全曲埋め込み
[ ] 16:9サムネを各曲紹介見出し直下に配置
[ ] WordPress下書き確認
[ ] 作業メモや内部注記を削除
[ ] SNS投稿文を作成
[ ] 告知漫画案を作成
```

## UI案

最初はシンプルな1画面でよい。

```text
1. 放送回選択
2. 記事タイプ選択
3. 作業範囲チェック
4. 音源整理設定
5. SE_Pon登録順プレビュー
6. 楽曲一覧
7. Codex作業開始パック生成ボタン
8. クリップボードコピー
```

## 実装メモ

- 最初から完全自動化しなくてよい。
- まずはCodexへ渡すテキスト生成を優先する。
- 音源ファイル保存やSE_Pon登録は、Codexが別作業として実行できるように、保存先と登録順を明確に出す。
- Radio-Article-Studioへ組み込む場合は、既存の放送回、楽曲、回答、サムネ情報を読み取ってパックを作る。
- 別ツールにする場合は、CSV/JSON貼り付けと手入力でパック生成できればよい。

## 別チャットでの開発依頼文

```text
Obsidianの以下ファイルを読んで、Sunoパ！作業開始パックツールを作ってください。

C:\Users\myabe\OneDrive\Desktop\Obsidian Folder\Umbrella Parade\Sunoパ！記事\11_Sunoパ作業開始パックツール_設計指示書.md

目的は、Radio-Article-Studioで得た放送回・楽曲・応募者・ゲスト・音源・サムネ情報をもとに、Codexへ渡す作業開始パックを生成することです。

まずは完全自動化よりも、作業開始パックのテキスト生成、SE_Pon登録順、音源保存先、作業後チェックリストの生成を優先してください。
```
