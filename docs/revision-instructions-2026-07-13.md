# Sunoパ！作業開始パックツール 修正指示書

対象URL:

```text
https://umbrellaparade.github.io/sunopa-work-start-pack-tool/
```

## 現状評価

ツールの方向性は合っています。べるぼ☂が毎回細かく操作する管理画面というより、Codexが作業前に必要情報を整えるための「作業開始パック生成ツール」として使います。

良い点:

- 基本情報、Radio-Article-StudioのCodexパック、作業範囲、音源保存、SE_Pon、楽曲一覧がまとまっている。
- 音源保存先が放送回名入りで具体化される。
- SE_Pon登録順がゲスト回の基本順どおりに生成される。
- `全曲リピートON: はい` が出る。
- 16:9サムネ制作指示が曲別に出る。
- 未入力時に不足情報を出せる。

## 修正したいこと

### 1. 生成前の右側にプレビュー枠を出す

生成前でも右側に、次のようなプレースホルダーを表示する。

```text
ここにCodexへ渡す作業開始パックが生成されます。
左側に放送回情報、Radio-Article-StudioのCodexパック、楽曲情報を入れて「パック生成」を押してください。
```

### 2. ボタン名を分かりやすくする

```text
上部ボタン: パック生成
生成結果側ボタン: 再生成
```

### 3. ツールの用途を明記する

画面上部のタイトル周辺に、小さく用途を入れる。

```text
Codex作業開始用
Radio-Article-Studioの情報を、記事作成・音源整理・SE_Pon登録の依頼文に整えるツール
```

### 4. テンプレート適用ボタンを追加する

追加したいボタン:

```text
ゲスト回テンプレを適用
リスナー回テンプレを適用
```

ゲスト回テンプレ:

```text
1. ゲスト曲
2. かなめ🦐曲
3. べるぼ☂曲
4. ゲスト曲
5. 応募者曲
```

リスナー回テンプレ:

```text
1. 応募者曲
2. 応募者曲
3. 応募者曲
4. 応募者曲
5. かなめ🦐曲
6. べるぼ☂曲
```

### 5. コピー成功表示を追加する

コピー成功時に短い表示を出す。

```text
コピーしました
```

### 6. 不足情報を画面上でも表示する

生成結果内の `確認してほしいこと` に加えて、画面上でも小さな注意ボックスとして表示する。

## 優先順位

優先度高:

1. 生成前の右側プレビュー枠
2. ボタン名を `パック生成` / `再生成` に変更
3. タイトル周辺に `Codex作業開始用` と用途説明を追加

優先度中:

4. ゲスト回テンプレ / リスナー回テンプレボタン
5. コピー成功表示
6. 不足情報の画面表示

## 変更しなくてよいこと

- 入力項目の大枠
- 音源保存先の生成
- SE_Pon登録順の生成
- `全曲リピートON`
- 楽曲紹介サムネの16:9指示
- 作業後チェックリスト
